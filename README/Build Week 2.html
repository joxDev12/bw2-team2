Progetto 01
CineTrack — Film Tracker
React Express PostgreSQL JWT

Gli utenti si registrano e costruiscono il proprio diario cinematografico: segnano i film visti, scrivono recensioni con un voto e tengono una watchlist di titoli da vedere. Tutti i dati dei film sono inseriti manualmente (non è necessaria un'API esterna).

Funzionalità richieste
  • Registrazione e login con JWT
  • CRUD completo sui film (titolo, anno, genere, regista, poster URL)
  • Aggiungi un film a "visto" o "watchlist"
  • Scrivi una recensione con voto 1–5 ⭐
  • Pagina profilo: film visti, media voti dati, watchlist
  • Lista pubblica di tutti i film con media voti
  • Pagina dettaglio film con tutte le recensioni
Schema database
TabellaCampi principali
usersid, email, password_hash, username
moviesid, title, year, genre, director, poster_url
reviewsid, user_id→, movie_id→, rating, body
watchlistid, user_id→, movie_id→, status (watched/to_watch)
Route backend minime
  • POST /auth/register · POST /auth/login
  • GET/POST/PUT/DELETE /movies
  • GET/POST/DELETE /reviews
  • GET/POST/DELETE /watchlist
  • GET /users/:id/profile
Bonus challenge

Feed globale "ultime recensioni" · Filtro per genere nella lista film · Statistiche globali (film più visti, meglio votati) · Paginazione dei risultati

Progetto 02
EventHub — Gestione eventi
React Express PostgreSQL JWT Ruoli

Due tipologie di utenti: Organizzatori che creano e gestiscono eventi, e Partecipanti che sfogliano il catalogo e si iscrivono. L'organizzatore vede chi si è iscritto; il partecipante vede lo stato delle proprie iscrizioni.

Funzionalità richieste
  • Registrazione con scelta del ruolo
  • Organizzatore: crea / modifica / cancella eventi
  • Evento: titolo, descrizione, data, luogo, capienza max, categoria
  • Partecipante: iscriviti a un evento (max capienza)
  • Partecipante: cancella la propria iscrizione
  • Organizzatore: lista iscritti per ogni evento
  • Dashboard personale differente per ruolo
Schema database
TabellaCampi principali
usersid, email, password_hash, username, role
eventsid, organizer_id→, title, date, location, max_seats, category
registrationsid, user_id→, event_id→, registered_at
Route backend minime
  • POST /auth/register · POST /auth/login
  • GET /events · GET /events/:id
  • POST/PUT/DELETE /events (solo organizzatori)
  • POST/DELETE /registrations
  • GET /events/:id/registrations (solo organizzatori)
Bonus challenge

Filtro per categoria e data · Countdown all'evento · Evento "completo" quando si raggiunge la capienza · Admin panel per moderare eventi

Progetto 03
JobBoard — Bacheca offerte lavoro
React Express PostgreSQL JWT Ruoli

Piattaforma per il mercato del lavoro con due attori: Aziende che pubblicano offerte, e Candidati che si candidano con un messaggio di presentazione. Le aziende gestiscono le candidature ricevute.

Funzionalità richieste
  • Registrazione con scelta del ruolo (azienda / candidato)
  • Azienda: crea / modifica / elimina annunci
  • Annuncio: titolo, descrizione, tipo contratto, città, stipendio
  • Candidato: invia candidatura con lettera di presentazione
  • Candidato: vede la lista delle proprie candidature e stato
  • Azienda: vede lista candidati per ogni offerta
  • Azienda: aggiorna stato candidatura (in revisione / accettata / rifiutata)
Schema database
TabellaCampi principali
usersid, email, password_hash, name, role
job_listingsid, company_id→, title, description, contract_type, city, salary
applicationsid, candidate_id→, job_id→, cover_letter, status
Route backend minime
  • POST /auth/register · POST /auth/login
  • GET /jobs · GET /jobs/:id
  • POST/PUT/DELETE /jobs (solo aziende)
  • POST /applications · GET /applications/mine
  • GET /jobs/:id/applications (solo aziende)
  • PATCH /applications/:id/status (solo aziende)
Bonus challenge

Filtri per città e tipo contratto · Salva offerta nei preferiti · Ricerca per parola chiave nel titolo · Profilo azienda con tutte le offerte attive

Progetto 04
RecipeBook — Community di ricette
React Express PostgreSQL JWT

Piattaforma dove gli utenti condividono le proprie ricette di cucina. Ogni ricetta ha una lista strutturata di ingredienti con quantità, e passi di preparazione ordinati. Gli altri utenti possono salvarle nei preferiti.

Funzionalità richieste
  • Registrazione e login con JWT
  • Crea / modifica / elimina le proprie ricette
  • Ricetta: titolo, descrizione, categoria, tempo prep, difficoltà
  • Ingredienti con quantità e unità (relazione separata)
  • Passi di preparazione ordinati (step 1, 2, 3…)
  • Salva / rimuovi ricette dai preferiti
  • Filtra ricette per categoria
  • Pagina profilo con le proprie ricette e i preferiti
Schema database
TabellaCampi principali
usersid, email, password_hash, username
recipesid, author_id→, title, category, prep_time, difficulty
ingredientsid, recipe_id→, name, quantity, unit
stepsid, recipe_id→, step_number, description
favoritesid, user_id→, recipe_id→
Route backend minime
  • POST /auth/register · POST /auth/login
  • GET /recipes · GET /recipes/:id
  • POST/PUT/DELETE /recipes (solo autore)
  • GET/POST/DELETE /favorites
  • GET /users/:id/profile
Bonus challenge

Commenti sulle ricette · Voto medio per ricetta · Ricerca per ingrediente · Upload immagine copertina (multer) · Calcolo porzioni dinamico nel frontend
