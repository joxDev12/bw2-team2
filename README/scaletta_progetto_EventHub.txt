3 tabelle

nome database EventHub

users, events, registration


2 ruoli: organizer, partecipant

in tabella eventi organizer_id puo essere solo un id di un user con ruolo organizer,
invece in tabella registrations user_id puo essere solo user con ruolo partecipant

in max_seats sara in integer che quando un partecipant si iscrive allora quel integer decrementa e
se annulla l'iscrizione aumenta. se l'integer e' 0 allora non e' possibile creare una registrazione a
quell'evento.
location sara' un varchar(500) per contenere indirizzo, citta, cap.

organizer_id e' FK di ID in tabella users.

anche user_id di registrations sara' Fk di ID in tabella users,
event_id sara' FK di ID in tabella events

un user puo prendere SOLO 1 posto per ogni evento!
quindi ID di registrations sara' la combinazione di user_id + event_id. quindi non ammettiamo duplicati

capisce dove mettere le UNIQUE

in tabella users role deve essere impostato con enum(organizer, partecipant)
